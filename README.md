IFT a.k.a. Instructor Forecast Tool was my very first Spreadsheet project that I've done in my life.										
If not this tool I probably woudn't have learned how to use Excel or even code in VBA (VisualBasic) as I do today.										

It all started with a version 1.0 which was really rought around the edges.										

<img width="1079" height="688" alt="image" src="https://github.com/user-attachments/assets/158185b2-1dac-4ee1-8e6c-a971cd809886" />

It was quite difficult to use but at the end of the day it did what it was designed to do.
Since it was pulling the data from multiple sources (Order Allocation, Day 0 Register & Day 1 Register)	
It was just a matter of time when it stops working.	

<img width="1070" height="619" alt="image" src="https://github.com/user-attachments/assets/8f079d69-b1d7-403a-96eb-7071b9bf93e8" />

Because of that I started to work on possible improvements (such as: easier user interface and better layout)
And then this version 2.0 was born:	

<img width="1079" height="774" alt="image" src="https://github.com/user-attachments/assets/2598aaa4-10f6-4efd-9225-d65b9c4dd8aa" />

But backend was still too complicated and it was still unoptimized.	

<img width="1079" height="906" alt="image" src="https://github.com/user-attachments/assets/dbbe1fe1-270f-46ff-9935-56e97bd62018" />

And Finally.										
Version 3.0 was born !	

<img width="1092" height="926" alt="image" src="https://github.com/user-attachments/assets/30d245fb-3239-4fb7-bfad-fb663c0fba75" />

Since It is fully cell formula based, there was almost no need for VBA code.										

<img width="1079" height="313" alt="image" src="https://github.com/user-attachments/assets/8a837b33-8bdb-4d26-8e52-08373cde47cc" />

Data has to be manually updated which is easy to do by following MANUAL, that meant less chances for failure.
What I learned creating this tool is that sometimes we make things more complicated than they need to be.
Simple = Better.

And that’s the story of this tool :)
